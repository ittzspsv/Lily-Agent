# Tool Formatters
### Introduction
- Provides a base abstraction for converting Tool objects into a standardized schema, used for LLM tool calling requests 

### Structure
```python
class Formatter(ABC):

    """ The only thing that you have to implement """

    @abstractmethod
    def format(self, tool: Tool) -> Dict[str, Any]:
        pass
```

### Purpose
- Formatter serves as the core abstraction for all the tool formatting implementations. 

## Creating our own tool formatter

### 2. Understanding the Tool Schema
- Before integrating tools with an LLM, it is crucial to understand what kind of schema structure is expected by the LLM.
- Tool schema defines how a tool is described.
#### Example 1
- The Tool Schema expected by OpenAI's LLMs (ChatGPT etc..)
```json
{
    "type": "function",
    "name": "get_horoscope",
    "description": "Get today's horoscope for an astrological sign.",
    "parameters": {
        "type": "object",
        "properties": {
            "sign": {
                "type": "string",
                "description": "An astrological sign like Taurus or Aquarius",
            },
        },
        "required": ["sign"],
    },
},
```
#### Example 2
- The tool schema expected by Anthropic's LLMs (Claude)
```json
{
    "name": "get_weather",
    "description": "Gets current weather for a location",
    "input_schema": {
        "type": "object",
        "properties": {
            "location": {
                "type": "string",
                "description": "City name"
            },
            "unit": {
                "type": "string",
                "enum": ["celsius", "fahrenheit"]
            }
        },
        "required": ["location"]
    }
}
```

### 3. Building our own Formatter
- Let us implement our own Anthropic's formatter by using **Example 2** as a reference.

```python
from lily_agent.formatters import Formatter
from lily_agent.tools import Tool
from typing import Dict, Any

class AnthropicFormatter(Formatter):
    def format(self, tool: Tool) -> Dict[str, Any]:
        """ This is the paramter schema generated by pydantic.  It's common for all. """
        schema = tool.input_schema.copy() 

        return {
            "name" : tool.name, # Accessing the tool name
            "description": tool.description, # Accessing the tool's description
            "input_schema": schema # The schema generated by pydantic
        }
```

### 4. Using the custom formatter.
- Let us use the custom formatter that we have built on our agent.
```python
from anthropic_formatter import AnthropicFormatter
from lily_agent import LilyAgent

agent = LilyAgent(
    formatter = AnthropicFormatter() # Define our formatter here.
    Adapter = OllamaAdapter(model="qwen2.5:7b"), # Not suitable but only for demonstration purpose only.
)

""" All other function just works fine """
```
